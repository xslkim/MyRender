顶点数据进来
经过顶点着色器做坐标变换

光栅化把三角形变成像素
片元着色器算出每个像素的颜色
最后写进帧缓冲显示出来
GPU 用硬件电路做这些
而我们
就用 C++ 把每一步老老实实模拟一遍

>>> 一帧是怎么画出来的 #B09

@exit: fade

--- visual ---
全屏深色背景 (#0d1117)。左侧竖排时间线，占画布左侧 55%，从上到下五个步骤节点，每个节点是一个圆角标签（背景 #161b22，accent 左边框 4px，高 90px，圆角 8px，文字 30px 白色），节点间用竖直 accent 虚线连接：
  "1. FrameStart — 清空颜色/深度缓冲"
  "2. 上传相机和光源到 gpu 全局变量"
  "3. 逐个物体：算模型矩阵 → Draw"
  "4. 逐三角形：顶点着色 → 裁剪 → 光栅化"
  "5. 颜色缓冲转 sRGB → 交给 SDL 显示"
右侧占画布 40%，展示 core/Scene.hpp 的 Render() 函数代码片段，等宽字体 24px，语法高亮（关键字 #ff7b72，字符串 #a5d6ff，注释 #8b949e）。
[0s] 右侧代码淡入。
[1s] 左侧五个节点从上往下依次滑入并高亮，间隔 0.6s。

--- narration ---
我们先建立一个全局的心理模型
每一帧
程序做这么五件事
第一步
清空颜色缓冲和深度缓冲
第二步
把相机和光源的数据
上传到一组全局变量里
第三步
遍历每个物体
算出它的模型矩阵
然后调用 Draw
第四步
对每个三角形
做顶点着色、裁剪、光栅化
第五步
把算好的颜色转成 sRGB
交给 SDL 显示到窗口上
这五步
就是第一集的地图
后面四集
我们会把第三步和第四步
彻底拆开讲
